## Summary

Implement the simplest autonomous agent sample: a single `QuestionAnswerer` agent that accepts a question via HTTP, processes it through an LLM decision loop, and returns a typed `Answer(answer, confidence)` result. The user submits a question via POST and polls for the result via GET.

## Component Design

### 1. Answer (domain record)

```java
// domain/Answer.java
public record Answer(String answer, int confidence) {}
```

No framework dependencies. Pure domain type.

### 2. QuestionTasks (task definitions)

```java
// application/QuestionTasks.java
public class QuestionTasks {
  public static final Task<Answer> ANSWER = Task
    .define("Answer")
    .description("Answer a question clearly and concisely, providing a confidence score")
    .resultConformsTo(Answer.class);
}
```

### 3. QuestionAnswerer (autonomous agent)

```java
// application/QuestionAnswerer.java
@Component(id = "question-answerer")
public class QuestionAnswerer extends AutonomousAgent {
  @Override
  public Strategy strategy() {
    return Strategy.autonomous()
      .goal("Answer questions clearly and concisely, showing reasoning step by step. " +
            "Provide a confidence score from 0 to 100.")
      .accepts(QuestionTasks.ANSWER)
      .maxIterations(3);
  }
}
```

No tools, no coordination. The simplest possible autonomous agent.

### 4. QuestionEndpoint (HTTP endpoint)

Two endpoints:
- `POST /questions` — accepts `QuestionRequest`, creates agent + task via `runSingleTask`, returns `QuestionResponse(taskId)`
- `GET /questions/{taskId}` — queries task snapshot, returns `AnswerResponse(answer, confidence, status)`

Handles edge cases:
- Empty/blank question → 400 Bad Request
- Unknown task ID → 404 Not Found
- Task still in progress → returns status "IN_PROGRESS" with null answer/confidence

### 5. QuestionAnswererIntegrationTest

Uses `TestModelProvider` with `fixedResponse()` to mock the LLM. Tests:
- Submit question and get task ID
- Poll and receive typed result with answer + confidence
- Verify task status transitions
