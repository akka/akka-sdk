# Implementation Plan: Consulting — Delegation + Handoff

**Branch**: `005-consulting` | **Date**: 2026-03-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-consulting/spec.md`

## Summary

Implement a consulting engagement system where a ConsultingCoordinator agent assesses problem complexity, delegates routine research to a ConsultingResearcher (retaining ownership), or hands off complex problems to a SeniorConsultant (transferring ownership). Demonstrates composing delegation and handoff in a single agent.

## Component Design

### 1. Domain records

```java
public record ConsultingResult(String assessment, String recommendation, boolean escalated) {}
public record ResearchSummary(String topic, String findings) {}
```

### 2. ConsultingTasks

```java
public class ConsultingTasks {
  public static final Task<ConsultingResult> ENGAGEMENT = Task
      .define("Engagement")
      .description("A consulting engagement to assess a client problem and deliver a recommendation")
      .resultConformsTo(ConsultingResult.class);

  public static final Task<ResearchSummary> RESEARCH = Task
      .define("Research")
      .description("Targeted research on a specific aspect of a consulting problem")
      .resultConformsTo(ResearchSummary.class);
}
```

### 3. ConsultingTools (shared)

```java
public class ConsultingTools {
  @FunctionTool(description = "Perform a preliminary assessment of a client problem")
  public String assessProblem(String problemDescription) { ... }

  @FunctionTool(description = "Check if a problem exceeds standard consulting scope and needs escalation")
  public String checkComplexity(String assessment) { ... }
}
```

`checkComplexity` returns "COMPLEX: ..." for regulatory/M&A/compliance, "STANDARD: ..." otherwise.

### 4. ConsultingCoordinator

```java
@Component(id = "consulting-coordinator")
public class ConsultingCoordinator extends AutonomousAgent {
  public Strategy strategy() {
    return Strategy.autonomous()
        .goal("Deliver actionable consulting recommendations. Assess each client " +
              "problem, determine its complexity, and ensure it reaches the right " +
              "level of expertise for resolution.")
        .accepts(ConsultingTasks.ENGAGEMENT)
        .tools(new ConsultingTools())
        .canDelegateTo(ConsultingResearcher.class)
        .canHandoffTo(SeniorConsultant.class)
        .maxIterations(10);
  }
}
```

### 5. ConsultingResearcher (delegation target)

```java
@Component(id = "consulting-researcher",
    description = "Performs targeted research on specific aspects of consulting problems")
public class ConsultingResearcher extends AutonomousAgent { ... }
```

Accepts `RESEARCH`, maxIterations(3).

### 6. SeniorConsultant (handoff target)

```java
@Component(id = "senior-consultant",
    description = "Handles complex, high-stakes consulting issues requiring senior expertise")
public class SeniorConsultant extends AutonomousAgent { ... }
```

Accepts `ENGAGEMENT`, maxIterations(5), tools(new ConsultingTools()).

### 7. ConsultingEndpoint

- `POST /engagements` — accepts `EngagementRequest(problem)`, creates coordinator + task via `runSingleTask`, returns `EngagementResponse(taskId)`
- `GET /engagements/{taskId}` — queries task snapshot, returns `ConsultingResultResponse`

### 8. Tests

Two key flows:
- **Standard problem**: Mock coordinator to delegate → mock researcher to complete → verify escalated=false
- **Complex problem**: Mock coordinator to hand off → mock senior to complete → verify escalated=true

Uses `TestModelProvider` with conditional responses per agent class.
